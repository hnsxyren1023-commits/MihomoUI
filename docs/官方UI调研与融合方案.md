# 官方 UI 调研与融合方案（MetaCubeXD）

## 1. 目标

在保留官方 UI 的同时，补齐“引导、诊断、启动效率”这三类体验能力，形成双 UI 协同：

- 官方 UI：深度管理与完整功能覆盖
- 自研 UI：轻量运维、状态可视化、故障定位

## 2. 官方方案优点

- 面板完整：代理组、连接、规则、日志等能力成熟
- 社区活跃：版本迭代快，生态成熟
- 对 controller API 覆盖广，适合长期使用

## 3. 官方方案常见痛点

- 后端不可达时，普通用户较难快速定位原因
- 初次配置路径、端口、Secret 时容易踩坑
- 部分场景下需要更轻量的“快速启停 + 诊断”入口

## 4. 融合策略

1. 保持官方 UI 原生体验，不改动核心逻辑  
2. 增加本地宿主层，用运行时配置注入后端地址  
3. 增加自研控制台，解决“启动前后排障”和“操作引导”  
4. 双 UI 共用同一 controller，避免配置割裂
